# Консольная утилита для парсинга и фильтрации файлов

## Задача, которую решает утилита
При запуске утилиты в командной строке подается несколько файлов, содержащих вперемешку целые числа, строки и вещественные числа.
В качестве разделителя используется перевод строки. Файлы и строки из них читаются по очереди в соответствии с их перечислением в командной строке.
Задача утилиты записать разные типы данных в разные файлы.
Целые числа в один выходной файл, вещественные в другой, строки в третий.

## Используемые технологии
- Java 17
- Apache Maven 3.4.2
- Lombok 1.18.38 (https://mvnrepository.com/artifact/org.projectlombok/lombok/1.18.38)

## Параметры утилиты
| Параметр      | Описание                              |
| ------------- | ------------------------------------- |
| `-o <path>`   | Путь для выходных файлов              |
| `-p <prefix>` | Префикс для имён выходных файлов      |
| `-a`          | Режим добавления в существующие файлы |
| `-s`          | Краткий отчёт                         |
| `-f`          | Полный отчёт                          |

Параметры можно задавать в любом порядке, результат будет одинаковый.<br>

Пример 1: `java -jar shift-java-test-1.0-SNAPSHOT.jar -s -a -p sample- file1.txt file2.txt`<br>

Пример 2: `java -jar shift-java-test-1.0-SNAPSHOT.jar -s file1.txt file2.txt -a -p sample-`

## Инструкция по запуску

### Запуск через командную строку
1. Скачать последний релиз сборки (https://github.com/KrosteInvise/shift-java-test/releases) 
2. Установить JDK, если еще не установлено (проверить можно открыв командную строку и написать java)
3. Открыть командую строку (желательно от имени администратора, чтобы не было проблем с правами для записи файлов)
4. Ввести `java -jar <путь до jar файла сборки> <параметры запуска>`

Пример: `java -jar shift-java-test-1.0-SNAPSHOT.jar -f -o C:\ -p TestPrefix_ file1.txt file2.txt`
